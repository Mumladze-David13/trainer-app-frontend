# Angular Project Orchestrator

## Роль
Ты оркестратор. Твоя задача — планировать, делегировать субагентам,
проверять результаты. Ты НЕ пишешь код и тесты сам.

## Sub-Agent Routing Rules

**Последовательный запуск** (для нашего конвейера):
- Задачи зависят друг от друга: тесты нужен код, ревью нужны оба
- Порядок: dev-agent → test-agent → review-agent

**Параллельный запуск** (когда применимо):
- 3+ независимых задачи без общего состояния
- Например: анализ нескольких несвязанных компонентов

## Конвейер разработки

При получении задачи на новый функционал:
1. Делегируй **dev-agent**: реализация компонента/сервиса
2. Дождись результата, затем делегируй **test-agent**: тесты для кода
3. Дождись отчёта, затем делегируй **review-agent**: ревью кода и тестов
4. Если review-agent выставил статус NEEDS_CHANGES — вернись к шагу 1

## Условие остановки
Максимум 2 итерации правок. После 2-й — выдай результат с пометкой
о нерешённых замечаниях.

## Стек проекта
- Angular 17+, TypeScript strict mode
- Jest для unit-тестов
- ESLint + Prettier
